雖然我有過寫遊戲的基礎，但是那些都是簡單的貪吃蛇，俄羅斯方塊，吃豆人。這些遊戲不要說什麼抗鋸齒，多重著色了，他們甚至連FPS都沒有。

但是，我相信，雖然我甚至連HELLOWORLD都沒有用JAVA寫過，更別提窗口。但我就是感覺，自己能夠做到。

碰撞檢測

2D中的碰轉檢測無非也就兩種寫法，多邊形檢測或者矩形檢測。我花費了不下兩個星期的時間，打算攻掠下多邊形檢測的算法，但是，我失敗了。我寫出的代碼，我自己都感覺不漂亮。不單單複雜度高，而且沒有任何的健壯性可言。我真的不敢把這樣的代碼放到我的遊戲引擎之中。

所以，最後，我使用了矩形檢測算法。也叫AABB。

這應該是世界上最簡單的碰轉檢測算法了。只需要一個return就可以完成。雖然很對不起自己的初衷，仕方ない。我真的沒有辦法在這麼短的時間之內，完成更多的事情了。

參考文獻

這個作業，我參考了JAVA2D遊戲設計，這本書寫得非常的好，但這不是一部面向小白的教程。我在看這本書的時候，也被一些JAVA有，C++沒有的特性或代碼的寫法整暈過。不過這都不是事。這本書除了教會我寫JAVA遊戲之外，還教會了我怎麼寫遊戲。

我想這大概會是我看的最後一本2D遊戲編程的書籍，因為，我感覺自己已經有了進入3D遊戲開發的能力。
